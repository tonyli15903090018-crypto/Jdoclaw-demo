# 基于OpenClaw的多端AI助手 - 产品需求文档

## 1. 项目定位

基于OpenClaw框架打造的AI对话助手,提供三个独立版本:
- **小程序版**(手机端)
- **Web版**(桌面端)
- **车机版**(车载中控屏)

其中Web版和车机版UI设计相同,仅在交互细节上有差异(车机版字号更大、默认语音输入、支持语音通话)。

---

## 2. 核心功能模块

### 2.1 用户注册与登录

**注册流程**:
1. 输入用户名和邮箱完成注册
2. 跳转至创建机器人页面
3. 输入机器人名称,选择机器人头像(emoji)
4. 创建完成后进入主应用

**登录**:
- 支持用户名/邮箱登录
- 登录状态持久化
- 退出登录功能

---

### 2.2 聊天模块

**基础对话**:
- 文本输入框,支持多行输入
- 发送消息给AI
- AI回复实时显示
- 区分用户消息和AI消息(不同气泡样式和头像)
- 显示消息时间戳
- AI思考时显示"正在输入"动画

**顶部提醒区域**:
- 优先展示每日贴士/今日任务
- 点击可跳转至日程管家查看详情
- 支持关闭/收起提醒

**对话界面**:
- 顶部功能栏:AI助手状态、用户名、主题切换、退出登录
- 顶部提醒:每日贴士/今日任务(优先展示)
- 中间消息区域:对话历史,支持滚动查看
- 底部输入区域:文本框+发送按钮

**车机端专属 - 语音输入**:
- 默认语音输入模式
- 支持切换至键盘输入
- 输入方式切换按钮

**车机端专属 - 语音通话**:
- 语音通话按钮(位于输入区域右侧)
- 点击发起语音通话
- 通话时顶部显示通话计时器(分:秒格式)
- 支持切换至悬浮窗状态(通话时可操作其他页面)
- 悬浮窗显示通话时长和挂断按钮
- 再次点击挂断通话

**模型选择**:
- 顶部有模型选择下拉框
- 可切换不同AI模型进行对话

---

### 2.3 路书模块

**功能说明**:
- 路书不是用户手动创建,而是通过聊天让AI生成
- 用户在聊天中表达旅行需求("我想去成都玩3天")
- AI返回结构化的路书数据
- 前端解析并展示路书内容

**路书生成流程**:
1. 用户在聊天中提出旅行需求
2. AI分析需求并生成路书
3. 路书自动保存到路书列表
4. 用户可在路书模块查看和管理

**路书列表**:
- 顶部统计卡片:
  - 总路书数量
  - 总里程数
  - 总天数
- 路书卡片展示:
  - 路书封面(emoji图标)
  - 路书标题
  - 行程距离和时长
  - 重点景点标签(最多3个)
  - 创建日期
  - 操作按钮:查看详情、编辑、删除

**路书详情**:
- 显示完整行程规划
- 每日行程安排
- 途经景点、餐厅、酒店等信息
- 预算估算

**车机端专属 - 路书导航**:
- 车机端在线时根据LBS(位置服务)判断当前位置
- 自动匹配正在进行的路书
- 显示当前进度(已到达/即将到达)
- 支持导入导航:
  - 点击"导航至下一站"
  - 调用车机导航系统(高德/百度等)
  - 自动输入目的地坐标

---

### 2.4 日程管家模块

**功能说明**:
- 日程管家展示OpenClaw返回的结构化任务数据
- 包含定时任务、今日任务、每日贴士等
- 前端做可视化展示和交互

**定时任务**:
- **任务创建**:通过聊天触发("每天早上8点提醒我晨跑")
- AI解析需求并创建定时任务
- 任务列表展示:
  - 任务名称
  - 执行计划(每天9:00/每周一等)
  - 开关按钮(启用/禁用)
  - 任务类型标签(提醒/自动执行等)

**定时任务历史查询**:
- 点击任务可查看历史执行记录
- 例如"每日生成新闻播客"任务:
  - 显示每日生成的播客列表
  - 点击可查看/收听具体内容
  - 显示生成时间和状态(成功/失败)

**今日任务**:
- 显示今日待办事项列表
- 支持勾选完成
- 显示任务名称和计划时间
- 完成后移至"已完成"区域

**每日贴士**:
- 系统推送的每日建议
- 显示健康提醒、生活小技巧等内容
- 同时在聊天页面顶部展示

**已完成任务**:
- 查看今日已完成的任务
- 显示完成标记(删除线)
- 显示完成时间

---

### 2.5 机器人部署模块

**支持平台**:
- QQ
- 飞书
- 企业微信

**渠道绑定**:
- 每个渠道显示:
  - 渠道图标和名称
  - 绑定状态(已绑定/未绑定)
  - 绑定时间
  - 绑定/解绑按钮

**部署指南**:
- 提供分步骤的部署教程
- 说明如何在QQ/飞书/企业微信创建机器人
- 如何获取API密钥
- 如何配置webhook地址

---

### 2.6 个人中心模块

**用户信息**:
- 显示用户头像(emoji)
- 显示用户名
- 显示手机号(可选绑定)
- 显示账户余额

**会员管理**:
- 显示会员状态(普通用户/会员)
- 会员加入时间
- **注意:会员一旦加入,永久有效,无需续费**
- 加入会员弹窗:
  - 会员权益说明
  - 一次性购买价格
  - 立即购买按钮
  - 支持微信支付和支付宝支付

**沙箱续期**:
- 显示沙箱到期时间
- 续期按钮
- 续期套餐选择(月度/季度/年度)
- 续期费用说明
- 支持微信支付和支付宝支付

**API充值**:
- 查看当前API余额
- 选择充值套餐(不同金额档位)
- 选择充值的API类型(不同AI模型)
- 显示充值后预计可用次数
- 显示费用明细
- 确认充值
- 支持微信支付和支付宝支付

**模型选择**:
- 查看所有可用AI模型列表
- 显示当前选中的模型
- 点击切换默认使用的模型

**使用记录**:
- 查看历史对话记录
- 每条记录显示:
  - 对话类型(文字/语音)
  - 使用的AI模型
  - 消耗的tokens数量
  - 费用金额
  - 对话时间

## 3. 三端适配需求

### 3.1 小程序版(手机端)

**使用场景**: 移动场景,随时随地使用

**布局特点**:
- 底部Tab导航:
  - 聊天
  - 路书
  - 日程管家
  - 机器人
  - 个人中心
- 单列纵向布局
- 紧凑信息密度,适配小屏幕(375px宽)
- 所有页面无需左右滚动

**交互特点**:
- 聊天页面:顶部功能栏固定,底部输入框固定,中间消息区域可滚动
- 路书页面:卡片垂直排列,封面横向展示
- 日程管家:任务列表垂直滚动
- 支持下拉刷新
- 支持微信登录
- 支持微信支付(入会/续期/充值)

**性能要求**:
- 首屏加载 < 2秒
- 页面切换流畅无卡顿

---

### 3.2 Web版(桌面端)

**使用场景**: 桌面浏览器访问

**布局特点**:
- 左侧边栏导航:
  - 聊天
  - 路书
  - 日程管家
  - 机器人
  - 个人中心
- 右侧主内容区域
- 多列网格布局(路书列表等)

**交互特点**:
- 支持键盘操作(Enter发送消息)
- 鼠标悬停效果
- 响应式设计,适配不同分辨率

**支付方式**:
- 支持微信支付(扫码)
- 支持支付宝支付(扫码)

---

### 3.3 车机版(车载中控屏)

**使用场景**: 车载中控屏,驾驶途中使用

**UI设计**: 与Web版相同,但有以下调整:
- **超大字号**: 所有文字放大(18px+)
- **超大按钮**: 所有可点击元素放大(48px+)
- **增大间距**: 按钮间距更大,防止误触

**专属功能**:

1. **语音输入(默认)**:
   - 车机端默认语音输入模式
   - 支持切换至键盘输入
   - 输入方式切换按钮

2. **语音通话**:
   - 语音通话按钮(位于输入区域)
   - 点击发起语音通话
   - 通话时顶部显示通话计时器
   - 支持切换至悬浮窗状态:
     - 通话时可切换至其他页面(路书/导航等)
     - 悬浮窗显示通话时长
     - 悬浮窗提供挂断按钮
     - 点击悬浮窗可回到聊天页面
   - 挂断通话

3. **路书导航**:
   - 根据LBS判断当前位置
   - 自动匹配正在进行的路书
   - 显示当前进度
   - 支持导入导航:
     - 点击"导航至下一站"
     - 调用车机导航系统
     - 自动输入目的地

**交互优化**:
- 简化操作流程
- 操作反馈更明显
- 按钮点击区域更大

**支付方式**:
- 支持微信支付(车机扫码/手机确认)
- 支持支付宝支付(车机扫码/手机确认)

## 4. 商业模式

**入会流程**:
1. 用户注册后为普通用户
2. 点击"加入会员"按钮
3. 选择支付方式(微信/支付宝)
4. 支付会员费用
5. 成为会员后**永久有效,无需续费**

**沙箱续期**:
- 会员需要定期续期沙箱环境
- 续期套餐:月度/季度/年度
- 沙箱到期前提醒用户续期
- 支持微信支付和支付宝支付

**API充值**:
- 使用AI对话需要消耗API额度
- 用户可充值购买API tokens
- 不同AI模型消耗不同tokens
- 余额不足时提示充值
- 支持微信支付和支付宝支付

**完整流程**:
```
注册 → 创建机器人 → 加入会员(一次性,微信/支付宝) → 购买沙箱(定期续期,微信/支付宝) → 充值API额度(按需购买,微信/支付宝) → 使用AI功能
```

## 6. 数据持久化需求

**需要保存的数据**:
- 用户登录状态
- 主题偏好设置
- 默认AI模型选择
- 对话历史记录(本地缓存)
- 用户信息(用户名、头像、余额等)
- 路书数据(AI生成的路书列表)
- 定时任务数据(任务列表和历史记录)

**云端同步**:
- 对话历史云端备份
- 路书数据云端存储
- 定时任务云端管理
- 跨设备同步设置

## 7. 核心交互流程

### 7.1 路书生成流程
1. 用户在聊天中提出旅行需求
   - 例:"我想去成都玩3天,推荐一下路线"
2. AI分析需求并调用路书Skill
3. 返回结构化路书数据
4. 前端解析并展示路书
5. 路书自动保存到路书列表
6. 用户可在路书模块查看和编辑

### 7.2 定时任务创建流程
1. 用户在聊天中提出定时需求
   - 例:"每天早上8点提醒我晨跑"
2. AI解析需求并创建定时任务
3. 任务保存到云端
4. 在日程管家模块显示任务
5. 到达设定时间时触发提醒
6. 用户可查看任务历史执行记录

### 7.3 车机端导航流程
1. 车机端检测到用户在行驶中
2. 根据LBS匹配正在进行的路书
3. 显示当前位置和下一站
4. 用户点击"导航至下一站"
5. 调用车机导航系统
6. 自动输入目的地坐标
7. 开始导航

### 7.4 车机端语音通话流程
1. 用户点击语音通话按钮
2. 发起与AI的语音通话
3. 通话计时器开始计时
4. 用户需要切换到导航页面
5. 点击"悬浮窗"按钮
6. 通话切换至悬浮窗状态
7. 用户可操作其他页面,悬浮窗显示通话状态
8. 点击悬浮窗可回到聊天页面
9. 点击挂断按钮结束通话
